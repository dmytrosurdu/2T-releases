# ω as a functor to BZ₂ from a presented “record transport” groupoid (Phase I)

This module defines a *presented groupoid* of “record processes” at fixed RG scale `u`
(Phase I: fixed `u`, no inter-scale morphisms). The groupoid is generated by:

* time steps `time t : r ↝ act t r`,
* a distinguished loop `loop r : r ↝ r` (the “square-loop” generator),
* composition, inverses, and type casts (transport of targets along equalities).

Relations include:

* category/groupoid laws (assoc/unit/inverses),
* time action laws (unit/comp/inv, up to casts),
* loop laws: involutive (`loop² = id`) and naturality/centrality (`time` commutes with `loop` up to change of basepoint).

We then define the parity map
[
\omega : \mathrm{Proc}(r,s) \to \mathbb{Z}_2 \cong \mathrm{Bool},
]
sending all time steps to `false`, loops to `true`, composition to xor, and inverses to identity.
Finally, we provide a “functor-to-BZ₂” wrapper and a proof that `ω` respects the presented relations.

> **Team note**
>
> * This file does *not* construct `loop` from SK squares. That belongs to the integration layer
>   once `SquareIso` on `Cofiber diag` is available (Track B) and the square-loop syntax is fixed (Track A).
> * Here we only set the *presentation and ω* in a clean “no truncations” way.

## Z₂ = Bool with xor

We define xor and its core laws (by case analysis). These are used to prove that
ω respects associativity/identity/inverse relations in the presented groupoid.

```agda
not : Bool → Bool
not true  = false
not false = true

xor : Bool → Bool → Bool
xor false b = b
xor true  b = not b

xor-idl : ∀ b → xor false b ≡ b
xor-idl b = refl

xor-idr : ∀ b → xor b false ≡ b
xor-idr false = refl
xor-idr true  = refl

xor-self : ∀ b → xor b b ≡ false
xor-self false = refl
xor-self true  = refl

xor-comm : ∀ a b → xor a b ≡ xor b a
xor-comm false b = xor-idl b ∙ sym (xor-idr b)
xor-comm true  false = refl
xor-comm true  true  = refl

xor-assoc : ∀ a b c → xor (xor a b) c ≡ xor a (xor b c)
xor-assoc false b c = refl
xor-assoc true  false false = refl
xor-assoc true  false true  = refl
xor-assoc true  true  false = refl
xor-assoc true  true  true  = refl
```

## Record dynamics at fixed scale u (time action on records)

We parameterize by a group `Time` and a time action `act : T → R → R` on a record type `R`,
together with the coherence equalities needed to type the time relations.

```agda
record RecordDynamics {ℓT ℓR : Level} (Time : Group ℓT) : Set (lsuc (ℓT ⊔ ℓR)) where
  open Group Time renaming (Carrier to T; e to eT; _∙_ to _∙T_; inv to invT)
  field
    R : Set ℓR
    act : T → R → R

    -- Coherence laws (assumed, not proved here)
    act-id   : ∀ r → act eT r ≡ r
    act-comp : ∀ r (t₁ t₂ : T) → act (t₂ ∙T t₁) r ≡ act t₂ (act t₁ r)
    act-inv  : ∀ r (t : T) → act (invT t) (act t r) ≡ r
```

## Presented record process groupoid (syntax + relations)

### Syntax of morphisms

We define a family `Proc r s` of morphism expressions between records.
It includes casts to adjust codomain along definitional equality proof terms.

```agda
module ΩInst {ℓT ℓR : Level} (Time : Group ℓT) (RD : RecordDynamics {ℓT} {ℓR} Time) where
  open Group Time renaming (Carrier to T; e to eT; _∙_ to _∙T_; inv to invT)
  open RecordDynamics RD using (R; act; act-id; act-comp; act-inv) public

  infixr 30 _⨟_
  data Proc : R → R → Set (ℓT ⊔ ℓR) where
    id   : ∀ r → Proc r r
    time : ∀ {r : R} → (t : T) → Proc r (act t r)
    loop : ∀ r → Proc r r

    _⨟_  : ∀ {r s t : R} → Proc r s → Proc s t → Proc r t
    inv  : ∀ {r s : R} → Proc r s → Proc s r

    castR : ∀ {r s s' : R} → s ≡ s' → Proc r s → Proc r s'
```

### Presented equality (thin 2-cells as derivations)

```agda
  infix 2 _≈_
  data _≈_ : ∀ {r s : R} → Proc r s → Proc r s → Set (ℓT ⊔ ℓR) where
    ≈-refl  : ∀ {r s} {p : Proc r s} → p ≈ p
    ≈-sym   : ∀ {r s} {p q : Proc r s} → p ≈ q → q ≈ p
    ≈-trans : ∀ {r s} {p q u : Proc r s} → p ≈ q → q ≈ u → p ≈ u

    -- Congruence
    ≈-cong-⨟ : ∀ {r s t} {p p' : Proc r s} {q q' : Proc s t} →
               p ≈ p' → q ≈ q' → (p ⨟ q) ≈ (p' ⨟ q')
    ≈-cong-inv : ∀ {r s} {p q : Proc r s} → p ≈ q → inv p ≈ inv q
    ≈-cong-castR : ∀ {r s s'} (e : s ≡ s') {p q : Proc r s} →
                   p ≈ q → castR e p ≈ castR e q

    ----------------------------------------------------------------
    -- Category/groupoid laws (presented)
    ----------------------------------------------------------------
    ≈-assoc : ∀ {r s t u} (p : Proc r s) (q : Proc s t) (h : Proc t u) →
              ((p ⨟ q) ⨟ h) ≈ (p ⨟ (q ⨟ h))

    ≈-idl   : ∀ {r s} (p : Proc r s) → (id r ⨟ p) ≈ p
    ≈-idr   : ∀ {r s} (p : Proc r s) → (p ⨟ id s) ≈ p

    ≈-invL  : ∀ {r s} (p : Proc r s) → (inv p ⨟ p) ≈ id s
    ≈-invR  : ∀ {r s} (p : Proc r s) → (p ⨟ inv p) ≈ id r

    ≈-inv-inv : ∀ {r s} (p : Proc r s) → inv (inv p) ≈ p
    ≈-inv-id  : ∀ {r} → inv (id r) ≈ id r
    ≈-inv-comp : ∀ {r s t} (p : Proc r s) (q : Proc s t) →
                 inv (p ⨟ q) ≈ (inv q ⨟ inv p)

    ----------------------------------------------------------------
    -- Cast laws
    ----------------------------------------------------------------
    ≈-castR-refl : ∀ {r s} (p : Proc r s) → castR refl p ≈ p

    ----------------------------------------------------------------
    -- Time action laws (typed using casts)
    ----------------------------------------------------------------
    ≈-time-id : ∀ (r : R) →
      time {r = r} eT ≈ castR (sym (act-id r)) (id r)

    ≈-time-comp : ∀ (r : R) (t₁ t₂ : T) →
      (time {r = r} t₁ ⨟ time {r = act t₁ r} t₂)
      ≈ castR (act-comp r t₁ t₂) (time {r = r} (t₂ ∙T t₁))

    ≈-time-inv : ∀ (r : R) (t : T) →
      inv (time {r = r} t)
      ≈ castR (act-inv r t) (time {r = act t r} (invT t))

    ----------------------------------------------------------------
    -- Loop laws: involution and naturality/centrality
    ----------------------------------------------------------------
    ≈-loop² : ∀ (r : R) → (loop r ⨟ loop r) ≈ id r

    ≈-loop-nat : ∀ (r : R) (t : T) →
      (time {r = r} t ⨟ loop (act t r)) ≈ (loop r ⨟ time {r = r} t)
```

## ω on morphism syntax

ω counts loop parity mod 2 and ignores time/casts/inverses.

```agda
  ω : ∀ {r s : R} → Proc r s → Bool
  ω (id r)        = false
  ω (time t)      = false
  ω (loop r)      = true
  ω (p ⨟ q)       = xor (ω p) (ω q)
  ω (inv p)       = ω p
  ω (castR e p)   = ω p
```

---

## ω respects the presented relations (main Phase I lemma)

This is the key well-definedness statement: ω is constant on `≈`-equivalence classes.

We provide the proof by recursion on the derivation, using the xor laws above.

```agda
  ω-resp : ∀ {r s : R} {p q : Proc r s} → p ≈ q → ω p ≡ ω q
  ω-resp ≈-refl = refl
  ω-resp (≈-sym h) = sym (ω-resp h)
  ω-resp (≈-trans h₁ h₂) = ω-resp h₁ ∙ ω-resp h₂

  ω-resp (≈-cong-⨟ {p = p} {p' = p'} {q = q} {q' = q'} h₁ h₂) =
    cong (λ b → xor b (ω q)) (ω-resp h₁) ∙ cong (xor (ω p')) (ω-resp h₂)

  ω-resp (≈-cong-inv h) = ω-resp h
  ω-resp (≈-cong-castR e h) = ω-resp h

  -- Category/groupoid laws: reduce to xor laws
  ω-resp (≈-assoc p q h) =
    xor-assoc (ω p) (ω q) (ω h)

  ω-resp (≈-idl p) =
    -- ω(id ⨟ p) = xor false ω(p) = ω(p)
    xor-idl (ω p)

  ω-resp (≈-idr p) =
    -- ω(p ⨟ id) = xor ω(p) false = ω(p)
    xor-idr (ω p)

  ω-resp (≈-invL p) =
    -- ω(inv p ⨟ p) = xor ω(p) ω(p) = false = ω(id)
    xor-self (ω p)

  ω-resp (≈-invR p) =
    xor-self (ω p)

  ω-resp (≈-inv-inv p) = refl
  ω-resp ≈-inv-id = refl

  ω-resp (≈-inv-comp p q) =
    -- ω(inv(p⨟q)) = ω(p⨟q) = xor ωp ωq
    -- ω(inv q ⨟ inv p) = xor ωq ωp
    xor-comm (ω p) (ω q)

  -- Cast
  ω-resp (≈-castR-refl p) = refl

  -- Time laws: ω(time ...) = false, ω(cast ... id) = false
  ω-resp (≈-time-id r) = refl
  ω-resp (≈-time-comp r t₁ t₂) = refl
  ω-resp (≈-time-inv r t) = refl

  -- Loop laws
  ω-resp (≈-loop² r) =
    -- ω(loop⨟loop) = xor true true = false
    refl

  ω-resp (≈-loop-nat r t) =
    -- ω(time⨟loop) = xor false true = true
    -- ω(loop⨟time) = xor true false = true
    refl
```

> **Team note**
>
> The last two `refl` for loop laws work because `xor false true` and `xor true false`
> both reduce to `true`, and `xor true true` reduces to `false`. If xor is redefined,
> you may want to replace these with small lemmas for robustness.

## Functor-to-BZ₂ wrapper (presented)

Since `BZ₂` has one object and `Bool` morphisms with composition `xor`,
a functor is essentially a map on morphisms preserving:

* `id` ↦ `false`,
* composition ↦ `xor`,
* inverses ↦ identity,
  and respecting `≈`.

All of these are definitional for ω, except `≈`-respect which is `ω-resp`.

```agda
  record FunctorToBZ₂ : Set (lsuc (ℓT ⊔ ℓR)) where
    field
      onHom  : ∀ {r s : R} → Proc r s → Bool
      resp≈  : ∀ {r s} {p q : Proc r s} → p ≈ q → onHom p ≡ onHom q

      onId   : ∀ (r : R) → onHom (id r) ≡ false
      onComp : ∀ {r s t} (p : Proc r s) (q : Proc s t) →
               onHom (p ⨟ q) ≡ xor (onHom p) (onHom q)
      onInv  : ∀ {r s} (p : Proc r s) → onHom (inv p) ≡ onHom p

  ωFunctor : FunctorToBZ₂
  FunctorToBZ₂.onHom  ωFunctor = ω
  FunctorToBZ₂.resp≈  ωFunctor = ω-resp
  FunctorToBZ₂.onId   ωFunctor r = refl
  FunctorToBZ₂.onComp ωFunctor p q = refl
  FunctorToBZ₂.onInv  ωFunctor p = refl
```
